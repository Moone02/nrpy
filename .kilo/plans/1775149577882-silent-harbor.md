# Plan: Update Geodesics Scripts Import Style to Match Reference Pattern

## Scope

**Only import style fixes** -- change `from nrpy.helpers.parallelization.utilities import X` to `import nrpy.helpers.parallelization.utilities as parallel_utils` and update all call sites to use the `parallel_utils.` prefix.

**No changes to:**
- `utilities.py` itself
- Parallelization patterns (`#pragma omp parallel for`, CUDA thread indexing, etc.)
- Any logic or behavior

## Differences Found

### Import Style (12 files)

**Current (geodesics files):**
```python
from nrpy.helpers.parallelization.utilities import (
    generate_kernel_and_launch_code,
)
# or
from nrpy.helpers.parallelization.utilities import generate_kernel_and_launch_code, get_commondata_access
```

**Target (matching rhs_eval.py, sqrt_detgammahat_d3xx_volume_element.py, cahdprefactor_gf.py):**
```python
import nrpy.helpers.parallelization.utilities as parallel_utils
```

### Function Call Style

**Current:**
```python
prefunc, launch_body = generate_kernel_and_launch_code(...)
cd_access = get_commondata_access(parallelization)
```

**Target:**
```python
prefunc, launch_body = parallel_utils.generate_kernel_and_launch_code(...)
cd_access = parallel_utils.get_commondata_access(parallelization)
```
